
# Multi Stage Container

project example : [link](https://github.com/alfian-putra/tesJenkins.git/)

In a multistage Dockerfile, developers define multiple build stages, each encapsulating a specific set of instructions and dependencies (Geeks for Geeks). 

In this example I using Go project to demonstrate how multi stage work. This project can be found in the repository above. The content of Dockerfile :

```docker
# build stage
## using image golang:1.25-alpine and give it name build
FROM golang:1.25-alpine AS build
## Determine directory in the container to run the next step 
WORKDIR /app
# copy from local to container
COPY . ./
# download go dependency
RUN go mod download
# build project
RUN go build -o main .

# run stage
## using image alpine:3.19 and give it name production
FROM alpine:3.19 AS production
## Determine directory in the container to run the next step 
WORKDIR /app
## copy from previous container to current container (notice --from=<previous container name>)
COPY --from=Build /app/main /app/.env .
## Give hos access to port
EXPOSE 5000
## Run file
CMD ["./main"]
```
