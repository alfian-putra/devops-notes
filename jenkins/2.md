# Jenkins CICD From Github to Docker Container

project example : [link](https://github.com/alfian-putra/tesJenkins.git/)

In this project i explore how Jenkins CICD work. THe idea of this project :
- Using Docker Compose to setup backend service (Using Dockerfile) and MongoDB
- Add Jenkinsfile and run the docker compose on Jenkins

**Dockerfile :**
```docker
# build stage
## using image golang:1.25-alpine and give it name build
FROM golang:1.25-alpine AS build
## Determine directory in the container to run the next step 
WORKDIR /app
# copy from local to container
COPY . ./
# download go dependency
RUN go mod download
# build project
RUN go build -o main .

# run stage
## using image alpine:3.19 and give it name production
FROM alpine:3.19 AS production
## Determine directory in the container to run the next step 
WORKDIR /app
## copy from previous container to current container (notice --from=<previous container name>)
COPY --from=Build /app/main /app/.env .
## Give hos access to port
EXPOSE 5000
## Run file
CMD ["./main"]
```

**compose.yml**
```yml
services:
  mongo:    ## mongodb services
    image: mongo:7.0    ## using image mongo version 7.0
    restart: always ## containers start automatically when they exit, or when Docker restarts.
    ports: ## make prot accessible from host and another containter
      - 27017:27017
    networks: # using network backend
      - backend 
  webapp: # backend service
    build: # build dockerfile
      context: ./backend ## workdir to run Dockerfile
      dockerfile: Dockerfile ## dockerfile name
      target: Production ## container that will be used are container with name production (from Dockerfile)
    depends_on: ## Depends on service mongo
      - mongo
    ports: ## make prot accessible from host and another containter
      - 5000:5000
    networks: # using network backend, same network as mongo
      - backend
networks: # declaring network
  backend:
```

**Jenkins**
```groovy
pipeline {
    agent any

    stages {
        stage('Build') {
            steps {
                // run docker compose
                // note : we can add another segment like docker ps to show container runnning
                sh 'docker compose up -d'
            }
        }
    }
}
```

**Note :**
1. user jenkins should be a part of group docker
2. port 5000 and 27017 (container) should be not in use by another service
3. Whe we want to configure `GitHub hook trigger for GITScm polling` (for example to triger every pull request) github server should be send webhook to jenkins server thats mean if we using public github our jenkins server also should be on public network.
